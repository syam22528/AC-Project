GIFT-64-128 OPTIMIZATIONS IMPLEMENTED
=====================================

This file documents implemented GIFT optimizations (CPU + GPU + mode utilities).
It also verifies the requested optimization checklist.

1) REQUESTED CHECKLIST STATUS
=============================

1.1 S-box (4-bit SubCells)
--------------------------

- Lookup-table S-box on GPU: IMPLEMENTED
	- `sbox_layer_table_dev` uses a 16-entry S-box from constant memory.

- Bitsliced S-box logic on GPU: IMPLEMENTED
	- `sbox_layer_bitsliced_dev` applies fixed boolean equations (AND/XOR/NOT-style logic over bit lanes).

- Recommendation to use bitsliced S-box for GPU speed: PARTIALLY APPLIED
	- Bitsliced variant is implemented and benchmarkable.
	- Current default GPU variant is still `table` (bitsliced is available via `variant="bitsliced"`).


1.2 Permutation Layer (PermBits)
--------------------------------

- Per-round bitwise PermBits in GPU kernel: IMPLEMENTED
	- `p_layer_dev` permutes bits with shift/OR bitwise operations each round.
	- Destination indices come from a constant-memory P-box map.


1.3 AddRoundKey and Key Schedule
--------------------------------

- XOR-based AddRoundKey: IMPLEMENTED
	- Both CPU and GPU apply round XOR masks in each round.

- Precompute round key material for kernel use: IMPLEMENTED
	- `generate_round_masks(key)` computes all per-round masks once per key.

- Keep round masks close to compute path: IMPLEMENTED
	- GPU uploads masks once and stages them into shared memory at kernel start.


1.4 CPU Baseline Strategy
-------------------------

- Numba-optimized CPU baseline: IMPLEMENTED
	- CPU path is JIT-compiled and parallelized with `prange`.

- Table and bitsliced CPU options: IMPLEMENTED
	- CPU supports both `table` and `bitsliced` S-box variants.

- Python fallback baseline: NOT USED IN CURRENT PATH
	- Current CPU implementation requires Numba JIT and raises if unavailable.


2) ALL IMPLEMENTED OPTIMIZATIONS
================================

2.1 SubCells (S-box) Optimizations
----------------------------------

- Dual S-box strategies (table + bitsliced)
	- Both CPU and GPU provide table-based and bitsliced S-box execution paths.
	- Benefit: supports performance/ablation comparison and architecture-aware tuning.

- Constant-memory table S-box on GPU
	- 16-entry S-box table is read through constant memory in table variant.
	- Benefit: tiny LUT with efficient cached broadcast semantics.

- Bitsliced boolean-network S-box
	- Uses fixed bit equations over all nibble lanes in a 64-bit state word.
	- Benefit: table-free nonlinear layer with high arithmetic intensity.


2.2 PermBits Optimizations
--------------------------

- Bitwise permutation in registers
	- Permutation is computed with bit extract/shift/OR operations, no large data structures.
	- Benefit: predictable, branch-free per-round permutation logic.

- Constant-memory P-box map on GPU
	- P-box indices are provided from constant memory.
	- Benefit: low-overhead access to fixed permutation mapping.


2.3 AddRoundKey and Key Schedule Optimizations
----------------------------------------------

- Precomputed per-round XOR masks
	- Key schedule is transformed into 28 round masks before encryption loops.
	- Benefit: avoids recomputing key schedule operations inside hot per-block loops.

- Round-mask caching on CPU object
	- CPU caches generated round masks for the active key.
	- Benefit: eliminates repeated key-schedule generation for same-key workloads.

- Shared-memory round keys on GPU
	- GPU kernels copy round masks into shared memory once per thread block.
	- Benefit: lower repeated global-memory latency during rounds.


2.4 Kernel/Execution Optimizations
----------------------------------

- Thread coarsening on GPU
	- Each CUDA thread processes 2 blocks.
	- Benefit: improved occupancy/latency hiding and reduced launch overhead per block.

- Parallel block processing on CPU
	- Numba `parallel=True` + `prange` distributes independent blocks across workers.
	- Benefit: scalable multi-core throughput.

- Worker clamping to available cores
	- Requested CPU workers are capped to logical core count.
	- Benefit: avoids oversubscription overhead.


2.5 Data Layout and Conversion Optimizations
--------------------------------------------

- Packed 64-bit state representation
	- Blocks are converted to big-endian `u64` arrays for round processing.
	- Benefit: compact state form and efficient bit-level operations.


2.6 Mode and Utility Optimizations
----------------------------------

- First-class CTR mode support
	- CPU and GPU both expose `encrypt_ctr` implemented as ECB(counter) + XOR.
	- Benefit: consistent, reusable mode API for both backends.

- Shared CTR helper utilities
	- Counter construction and XOR are centralized in `ctr_utils`.
	- Benefit: reduced code duplication and consistent CPU/GPU semantics.


2.7 Correctness/Robustness Optimizations
----------------------------------------

- Input validation on public APIs
	- Enforces block-size alignment and key length constraints.
	- Benefit: fail-fast behavior and safer integration.

- CPU/GPU cross-check in benchmark flow
	- Benchmarks compare CPU and GPU outputs and fail on mismatch.
	- Benefit: protects optimization work from silent correctness regressions.


2.8 Additional Non-Main-Component Optimizations
-----------------------------------------------

- Numba compilation cache on CPU hot functions
	- CPU kernels and helpers are decorated with `@njit(cache=True)`.
	- Benefit: avoids repeated recompilation across runs and speeds iterative testing.

- One-time warm-up before timing in benchmark flow
	- CPU and GPU are both run once before timed measurements in each config.
	- Benefit: reduces first-run JIT/setup skew in reported performance.

- Median-based timing aggregation
	- Benchmark reports median over multiple runs for CPU and GPU timings.
	- Benefit: lowers noise from outliers and gives more stable throughput comparisons.

- Consistent thread configuration lifecycle on CPU
	- CPU thread count is set for the run and restored afterward.
	- Benefit: predictable parallel behavior without persistent thread-state side effects.
